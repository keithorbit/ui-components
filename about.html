What is UI Components?

UI components is functional CSS component library. It is powered by a 2kb CSS file.

How does it work?

Download and include the CSS file then copy any of the HTML component code.

Why does this exist? Why make it?

Functional CSS is a great tool for designing in the browser. The focus of this component library is to help rapidly communicate UI designs in a development team. Moving from a whiteboard or sketches on paper to a browser can sometimes be a hurdle, many UI designers are comfortable having a middle step with Sketch/Photoshop etc. In my personal use, I want to cut out the middle man. After all, the products we are designing to reside in the browser.

What's new about this?

Functional CSS moves the burden of styling your application from CSS to HTML. This at first may seem like a terrible idea (see separation of concerns) but it really isn't. The most DRY code you can hope to get is to combine functional CSS with a templated front end view (.net/rails partials, react components etc.)

Everything in the end compiles back to raw HTML and CSS.
